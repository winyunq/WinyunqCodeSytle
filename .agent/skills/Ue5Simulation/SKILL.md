# Ue5Simulation Skill

## 仿真逻辑
- **数据驱动**：UE5 场景应通过读取 `bin/` 目录下的输出数据进行回放。
- **Actor 定义**：导航物体应继承自专用 Base 类，支持外部姿态数据（Quaternion/Euler）注入。

## 接口协议
- **TCP/UDP**：若需实时通信，协议定义需在 `WinyunqCore` 级别的文档中备案。
- **JSON 交互**：UE5 应具备解析生成的 `.json` 配置文件以初始化场景参数的能力。

## 可视化规范
- **坐标对齐**：注意 UE5 的左手坐标系与惯导常规右手坐标系的映射关系。
- **性能优化**：大规模粒子系统或高频刷新率下的资源管理。
